# Т2. Написание баг репорта

# Тестирование требований

## BUG-REQ-001 — Отсутствует явное указание места поиска
Описание:
В требованиях не уточняется, по каким полям должен выполняться поиск:

- по названию файла
- по содержимому файла
- по расширению
- по тегам или метаданным

Ожидаемый результат:
В требованиях должно быть явно указано, по каким параметрам выполняется поиск.

## BUG-REQ-002 — Отсутствует описание чувствительности к регистру во время поиска
Описание:
Требования не уточняют, должен ли поиск отличать:

- test
- Test
- TEST

Ожидаемый результат:
Требование должно уточнять поиск: регистронезависимый или чувствителен к регистру.

## BUG-REQ-003 — Отсутствует описание поведения системы, если оставить поле поиска пустым
Описание:
Требования не указывают что происходит, если пользователь нажимает поиск с пустым полем.

Возможные варианты поведения:

- отображаются все файлы
- не отображается ни 1 файла
- ничего не происходит
- выводится сообщение об ошибке

Ожидаемый результат:
Требование должно описывать поведение системы при пустом запросе.

## BUG-REQ-004 — Отсутствует требование, указывающее на максимально допустимое количество результатов поиска
Описание:
В требованиях не указано, есть ли лимит результатов и вид их вывода.

Ожидаемый результат:
Требование должно описывать ограничение очень большого количества результатов либо их способ отображения.
  
## BUG-REQ-005 — Отсутствует описание вида отображения файлов из корзины в результатах поиска и возможные действия над ними
Описание:
В требованиях не указано:

- будет ли показано, что файл находится в корзине
- можно ли открыть файл
- можно ли восстановить его из результатов поиска

Ожидаемый результат:
Требование должно описывать визуальное отображение и действия для таких файлов.

## BUG-REQ-006 — Отсутствует описание поведения системы при наличии спецсимволов в запросе
Описание:
В требовании о допустимых символах не уточняется, как обрабатываются символы: `*`, `?`, `/`, `\`, `"`.
В некоторых системах они используются как поисковые операторы.

Ожидаемый результат:
Требование должно описывать правила обработки специальных символов.

# Тестирование UI

## BUG-UI-001 — Ошибочный вывод результатов поиска при вводе спецсимволов из таблицы Unicod
Описание:
Требование "Результат работы функции - набор файлов и папок, удовлетворяющих запросу." не выполняется при использовании любых спецсимволов из таблицы Unicode
Шаги воспроизведения:

- Открыть Яндекс.Диск `https://disk.yandex.ru`
- Создать таблицу `test`
- Вернуться на вкладку `Яндекс Диск`
- Ввести в поле поиска символ `☃`
- Нажать кнопку найти

Ожидаемый результат:
В области результатов поиска появляется надпись "В вашем Диске не найдено файлов или папок по запросу <☃>"

Актуальный результат:
В области результатов поиска появляется файл test.xlsx

Окружение:
Кастомная сборка ПК на базе Windows 10
Браузер Opera GX LVL 7 127.0.5778.96

Критичность:
Minor

Приоритет:
Low

Актуальный результат:

# Тестирование API

## BUG-API-001 — Получение некорректного кода ответа при успешном создании папки (201 вместо 200)

Описание:
Требования "В случае успешного выполнения запроса получен ответ 200, папка появилась по нужному адресу в диске" выполняется не корректно. 
Вместо требуемого ответа 200 приходит ответ 201.

Предусловия:
- Открыто приложение "Postman"
- Добавлены поля в Headers в соответствии с требованиями
- Открыта вкладка Params

Шаги воспроизведения:

- Открыть страницу Яндекс диска `https://disk.yandex.ru/client/disk` в браузере
- Проверить отсутствие папки `test` в корневом каталоге
- В Postman'e добавить параметр `path` со значением `disk:/test`
- Нажать на кнопку `Send`

Ожидаемый результат:
В ответ пришёл код 200 и тело ответа
