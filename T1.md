# T1 — Написание тест-кейсов

# 1. Цель тестирования

Проверить корректность работы REST API метода получения статуса продукта.

API должно:

- корректно обрабатывать обязательные и необязательные параметры
- корректно выполнять авторизацию
- корректно обрабатывать комбинации параметров
- возвращать корректные HTTP статусы
- возвращать корректный JSON
- корректно обрабатывать ошибки

# 2. Объект тестирования

Структура запроса:

GET `/products/{productId}/status?authToken=&recalculate=&owner=&region=`

Параметры запроса:

| Параметр    | Тип     | Обязательность | Возможные значения                    |
| ----------- | ------- | -------------- | ------------------------------------- |
| authToken   | string  | обязательный   | строка из 16 символов (буквы и цифры) |
| recalculate | boolean | необязательный | true / false / null                   |
| owner       | string  | необязательный | Создатель / Пользователь / null       |
| region      | string  | необязательный | Северо-Запад / Сибирь / Поволжье      |

Особенности:

- параметры recalculate, owner и region необязательные
- параметры не коррелируют между собой

# 3. Ожидаемые ответы API

Успешный ответ:

HTTP 200

JSON
{
  "productStatus": 1
}

Допустимые значения:

- 1 : продукт готов
- 0 : продукт не готов

Ошибки:

- 401 : отсутствует или неверный authToken
- 404 : продукт не найден
- 500 : внутренняя ошибка сервера
  
Пример ошибки:

JSON
{
  "errorMessage": "Internal server error"
}

# 4. Области подвергаемые тестированию

Тестируются следующие области системы:

- REST API GET запрос
- endpoint `/products/{productId}/status?authToken=&recalculate=&owner=&region=`
- корректность обработки параметров запроса
- корректность авторизации через `authToken`
- корректность формирования JSON ответа
- обработка ошибок API
- обработка отсутствующих параметров
- обработка некорректных параметров
- обработка различных значений `productId`

# 5. Области не подвергаемые тестированию

В рамках текущего тестирования не проверяются:

- внутренняя бизнес-логика расчета статуса продукта
- производительность системы
- безопасность API (pentest)
- UI приложения
- база данных и её структура

# 6. Риски

| Риск                            | Описание                                       |
| ------------------------------- | ---------------------------------------------- |
| Недоступность тестового сервера | невозможно выполнить тестирование              |
| Отсутствие тестовых данных      | невозможно проверить разные статусы продуктов  |
| Изменение API                   | тест-кейсы могут устареть                      |
| Недостаточная документация      | возможна неправильная интерпретация требований |

# 8. Критерии начала тестирования

Тестирование может быть начато если:

- API endpoint развернут
- доступна тестовая среда
- есть доступ к системе авторизации
- подготовлены тестовые данные

# 9. Критерии приостановки тестирования

Тестирование приостанавливается если:

- API недоступно
- критические дефекты блокируют тестирование
- отсутствуют необходимые тестовые данные

# 10. Критерии возобновления тестирования

Тестирование возобновляется если:

- восстановлена работа API
- исправлены блокирующие дефекты
- предоставлены необходимые данные

# 11. Критерии окончания тестирования

Тестирование считается завершенным если:

- выполнены все тест-кейсы
- исправлены критические дефекты
- подготовлен отчёт о тестировании

# 12. Результаты тестирования

Результатом тестирования являются:

- отчёт о выполнении тест-кейсов
- список найденных дефектов
- отчёт о покрытии требований
- итоговый тестовый отчёт

# 13. Стратегия тестирования

Тестирование включает следующие направления:

13.1 Функциональное тестирование

Проверка корректности работы endpoint.

13.2 Проверка параметров

Проверяются:

- обязательный параметр authToken
- необязательные параметры
- комбинации параметров

13.3 Pairwise testing

Используется pairwise-подход для проверки комбинаций параметров.

Это позволяет покрыть большинство дефектов при меньшем количестве тестов.

13.4 Граничные значения

Для authToken проверяются:

- длина < 16
- длина = 16
- длина > 16
- только цифры
- только буквы
- спецсимволы

13.5 Негативное тестирование

Проверяются:

- неверные значения параметров
- отсутствующие параметры
- неправильный формат
- неожиданные значения

13.6 Проверка формата ответа

Проверяется:

- корректность JSON
- типы данных
- наличие обязательных полей
